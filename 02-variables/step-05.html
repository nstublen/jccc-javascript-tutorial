The "truthiness" comparisons can be avoided with different operators. 

            a === b;     // strict equality
            a !== b;     // strict inequality

These statements are true or false, based on the variable's type and value. 

            "5" == 5;   // true
            "5" === 5;  // false

In general, I would suggest using the strict comparison operators unless you are intentionally allowing the conversion. The conversion is convenient, for example, when analysing a numeric value entered into a text field.
