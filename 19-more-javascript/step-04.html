Exceptions

When a JavaScript error occurs, the browser will stop executing our JavaScript code. If we want to make sure our code continues executing, we can catch the errors and respond appropriately. 

        function calculateSalesTax(items) {
            var subtotal = 0;
            for (var index = 0; index < items.length; ++index) {
                subtotal += items[index];
            }
            return subtotal * 0.085;
        }
        
        var arrayOfItems; // Oops! We didn't assign a value.
        calculateSalesTax(arrayOfItems);

We can make our function more robust by checking items before we use it, but we can also use exception handling with a try-catch block. 

        function calculateSalesTax2(items) {
            var subtotal = 0;
            try {
                for (var index = 0; index < items.length; ++index) {
                    subtotal += items[index];
                }
            } catch(e) {
                alert("Someone passed bad data to calculateSalesTax2()!");
                subtotal = 0;
            }
            return subtotal * 0.085;
        }

It's also possible to generate an exception using the throw keyword. 

        function calculateSalesTax3(items) {
            if (!items) {
                // Returning 0 would be inaccurate...
                throw new Error("Invalid items argument!")
            }
            
            var subtotal = 0;
            for (var index = 0; index < items.length; ++index) {
                subtotal += items[index];
            }
            return subtotal * 0.085;
        }
